# slow-sql-design-docx

根据慢 SQL 分析 Excel、上线包里的实际索引 DDL、以及既有 Word 概要设计模板，生成或填写一份谨慎的慢 SQL 优化概要设计文档。

## 适用场景

- 已有中文行业系统风格的概要设计 DOCX 模板
- 已有生产慢 SQL 分析 Excel
- 已有 tar/zip/sql 上线包，里面包含新增索引和回退脚本
- 需要把“分析建议”和“实际实施范围”对齐，避免把未上线的优化写进设计文档

## 核心原则

- 以脚本中的 DDL 作为本次投产范围边界
- Excel 作为慢 SQL 背景、原因和优化建议证据
- 不覆盖原始模板，默认生成新 DOCX
- 不确定的需求编号、投产日期、负责人、系统中文名等字段保留为“待补充”
- 交付前检查 DOCX 结构、上线 SQL、回退 SQL 和模板残留

## 输出结果

通常会生成一份新的概要设计 DOCX，并在最终回复中说明：

- 文档路径
- 实际写入的索引数量和涉及表
- 哪些字段因材料不足而留空
- 已完成的结构校验
- 是否完成视觉渲染检查
