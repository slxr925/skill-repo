# wechat-computer-use-send-message

> 一个用于在 macOS 上通过 `computer-use` 稳定发送微信消息的 skill。

## 它解决什么问题

这个 skill 主要解决一个很实际的问题：

当通过 `computer-use` 操作桌面微信时，英文输入往往比较顺，但中文输入可能会出现“光标在输入框里、文字却没有真正进去”的情况。看起来像执行成功了，实际上消息并没有发出去。

这个 skill 的目标，就是把这类不稳定行为收敛成一条更可靠的发送路径。

## 适用场景

- 需要通过桌面自动化操作微信
- 需要给指定联系人或群聊发送消息
- 直接输入中文不稳定，或者已经失败过
- 希望用一套更稳的流程完成“粘贴并发送”

## 核心思路

这套 skill 的核心不是“盲发”，而是“先验证，再发送”。

它采用的可靠路径是：

1. 先确认当前微信窗口和目标聊天是否正确
2. 将要发送的文本放进系统剪贴板
3. 使用粘贴快捷键把内容送进微信输入框
4. 先确认草稿已经真实出现在输入框里
5. 再按回车发送
6. 最后以聊天气泡是否出现作为成功标准

## 为什么这个 skill 有价值

在桌面自动化里，最容易踩坑的不是“命令报错”，而是“命令没报错，但结果不对”。

这个 skill 把几条已经验证过的经验固定了下来：

- `type_text` 发英文可以，发中文不一定可靠
- AppleScript 运行成功，不代表微信真的收到了文本
- 只有在输入框里看见草稿，才应该继续发送
- 只有在对话区看到发送气泡，才算真正成功

如果你想通过 `computer-use` 给微信发消息，尤其是发中文，这个 skill 的价值就在于：它帮你绕开“不报错但没发出去”的假成功。
